# 🚀 网站上线详细步骤指南

## 📋 准备工作检查清单

在开始之前，请确认：
- ✅ 项目代码已完成
- ✅ 本地测试通过（运行 `npm run dev` 能正常访问）
- ✅ 有 GitHub 账号（如果没有，需要先注册）
- ✅ 已安装 Git（如果没有，需要先安装）

---

## 第一步：检查本地环境

### 1.1 检查 Node.js 版本

打开 PowerShell（按 `Win + X`，选择 "Windows PowerShell"），输入：

```bash
node -v
```

**要求：** Node.js 版本需要 18 或更高。如果版本太低或没有安装：
1. 访问 https://nodejs.org
2. 下载并安装 LTS 版本
3. 安装完成后重启终端

### 1.2 检查 Git 是否安装

在 PowerShell 中输入：

```bash
git --version
```

**如果没有安装 Git：**
1. 访问 https://git-scm.com/download/win
2. 下载 Windows 版本
3. 运行安装程序，全部选择默认选项
4. 安装完成后**重启电脑**（或重启终端）

### 1.3 配置 Git（首次使用）

在 PowerShell 中输入以下命令（替换成你的信息）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

例如：
```bash
git config --global user.name "张三"
git config --global user.email "zhangsan@example.com"
```

## 第二步：创建 GitHub 仓库

### 2.1 注册/登录 GitHub

1. 访问 https://github.com
2. 如果没有账号：
   - 点击右上角 "Sign up"
   - 填写用户名、邮箱、密码
   - 验证邮箱完成注册
3. 如果已有账号，直接登录

### 2.2 创建新仓库

1. 登录后，点击右上角头像旁边的 **"+"** 按钮
2. 选择 **"New repository"**
3. 填写仓库信息：
   - **Repository name（仓库名称）**：`toolify-website`（或你喜欢的名字）
   - **Description（描述）**：`AI Tools Directory Website`（可选）
   - **Visibility（可见性）**：
     - 选择 **Public**（公开）- 免费，任何人都能看到代码
     - 或选择 **Private**（私有）- 只有你能看到代码
4. **重要：不要勾选以下选项**
   - ❌ Add a README file
   - ❌ Add .gitignore
   - ❌ Choose a license
5. 点击 **"Create repository"** 按钮

### 2.3 复制仓库地址

创建完成后，你会看到一个页面，上面有仓库地址，类似：
```
https://github.com/你的用户名/toolify-website.git
```

**复制这个地址，稍后会用到！** 📋

## 第三步：上传代码到 GitHub

### 3.1 打开项目文件夹

在 PowerShell 中，输入：

```bash
cd D:\Projects\website-clone
```

### 3.2 初始化 Git 仓库

```bash
git init
```

你会看到：`Initialized empty Git repository in D:/Projects/website-clone/.git/`

### 3.3 添加所有文件到 Git

```bash
git add .
```

这个命令会把项目中的所有文件添加到 Git 暂存区。

### 3.4 提交代码

```bash
git commit -m "Initial commit - Toolify website"
```

你会看到类似这样的输出：
```
[main (root-commit) xxxxxxx] Initial commit - Toolify website
 X files changed, Y insertions(+)
```

### 3.5 连接到 GitHub 仓库

**重要：将下面的 `你的用户名` 替换成你的 GitHub 用户名！**

```bash
git remote add origin https://github.com/你的用户名/toolify-website.git
```

例如，如果你的用户名是 `zhangsan`，仓库名是 `toolify-website`，则输入：
```bash
git remote add origin https://github.com/zhangsan/toolify-website.git
```

### 3.6 重命名主分支为 main

```bash
git branch -M main
```

### 3.7 上传代码到 GitHub

```bash
git push -u origin main
```

**第一次上传会要求登录：**

#### 情况 A：弹出浏览器窗口
- 会自动打开浏览器
- 登录你的 GitHub 账号
- 授权后会自动完成上传

#### 情况 B：要求输入用户名和密码
1. **用户名**：输入你的 GitHub 用户名
2. **密码**：**不能使用普通密码**，需要使用 Personal Access Token

**如何创建 Personal Access Token：**
1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 填写信息：
   - **Note（备注）**：`Git Push Token`
   - **Expiration（过期时间）**：选择 `90 days` 或 `No expiration`
   - **Select scopes（权限）**：勾选 `repo`（全部仓库权限）
4. 点击 **"Generate token"**
5. **重要：立即复制生成的 token**（只显示一次，关闭页面后就看不到了）
6. 在终端输入密码时，粘贴这个 token

### 3.8 验证上传成功

1. 刷新你的 GitHub 仓库页面（在浏览器中）
2. 应该能看到所有项目文件：
   - `package.json`
   - `src/` 文件夹
   - `public/` 文件夹
   - 等等
3. 如果看到文件列表，说明上传成功！✅

## 第四步：部署到 Vercel（推荐）

Vercel 是 Next.js 的官方推荐平台，部署非常简单。

### 4.1 访问 Vercel

1. 打开浏览器，访问：https://vercel.com
2. 点击右上角 **"Sign Up"** 或 **"Log In"**

### 4.2 使用 GitHub 登录

1. 点击 **"Continue with GitHub"** 按钮
2. 会跳转到 GitHub 授权页面
3. 点击 **"Authorize Vercel"** 按钮
4. 完成授权后，自动返回 Vercel 网站

### 4.3 创建新项目

1. 登录后，在 Vercel 首页点击 **"Add New..."** 按钮
2. 选择 **"Project"**
3. 你会看到你的 GitHub 仓库列表
4. 找到 `toolify-website` 仓库（或你创建的仓库名）
5. 点击仓库右侧的 **"Import"** 按钮

### 4.4 配置项目

Vercel 会自动检测到这是 Next.js 项目，配置如下：

```
Framework Preset: Next.js
Root Directory: ./
Build Command: npm run build
Output Directory: .next
Install Command: npm install
```

**通常不需要修改，直接使用默认设置即可！**

如果你需要修改：
- 点击 **"Edit"** 可以修改配置
- 但大多数情况下，默认配置就足够了

### 4.5 部署项目

1. 确认配置无误后，点击页面底部的 **"Deploy"** 按钮
2. 等待部署过程（通常 2-3 分钟）
3. 你会看到部署进度：
   - ⏳ Installing dependencies...（安装依赖）
   - ⏳ Building...（构建项目）
   - ⏳ Deploying...（部署中）

### 4.6 部署完成！

部署完成后，你会看到：
- ✅ 绿色的 "Success" 消息
- 🌐 你的网站地址，类似：`https://toolify-website.vercel.app`

**点击这个地址，就能访问你的网站了！** 🎉

## 第六步：自定义域名（可选）

如果你有自己的域名，可以绑定到 Vercel。

### 6.1 在 Vercel 中添加域名

1. 在 Vercel 项目页面，点击 **"Settings"** 标签
2. 点击左侧菜单的 **"Domains"**
3. 在输入框中输入你的域名（如：`www.toolify.ai` 或 `toolify.ai`）
4. 点击 **"Add"** 按钮

### 6.2 配置 DNS

Vercel 会显示需要添加的 DNS 记录，通常有两种情况：

**情况 A：使用 CNAME 记录（推荐）**
- 类型：`CNAME`
- 名称：`www`（或你想要的子域名）
- 值：`cname.vercel-dns.com`

**情况 B：使用 A 记录**
- 类型：`A`
- 名称：`@`（表示根域名）
- 值：Vercel 提供的 IP 地址

### 6.3 在域名服务商添加 DNS 记录

1. 登录你的域名服务商（如 GoDaddy、Namecheap、阿里云、腾讯云等）
2. 找到 DNS 管理或域名解析设置
3. 添加 Vercel 显示的 DNS 记录
4. 保存设置

### 6.4 等待 DNS 生效

- 通常需要 **几分钟到几小时**
- DNS 生效后，你的域名就能访问网站了
- 可以在 Vercel 的 Domains 页面查看状态

## 🔧 常见问题解决

### 问题 1：Git 命令找不到

**错误信息：** `git: command not found` 或 `'git' 不是内部或外部命令`

**解决方案：**
1. 确认已安装 Git
2. 重启终端或电脑
3. 如果还是不行，检查环境变量 PATH 是否包含 Git 的安装路径

### 问题 2：Git push 失败 - 认证错误

**错误信息：** `Authentication failed` 或 `remote: Invalid username or password`

**解决方案：**
1. 使用 Personal Access Token 代替密码（见第三步 3.7）
2. 或者使用 SSH 密钥（更安全，但设置较复杂）

### 问题 3：Vercel 部署失败

**可能原因和解决方案：**

1. **构建错误**
   - 点击 Vercel 部署日志查看详细错误
   - 检查 Node.js 版本（需要 18+）
   - 确保所有依赖都正确安装
   - 检查代码中是否有语法错误

2. **环境变量缺失**
   - 如果有环境变量，在 Vercel 项目设置 → Environment Variables 中添加

3. **路径错误**
   - 检查 `package.json` 中的脚本是否正确
   - 确认项目结构正确

### 问题 4：网站显示 404 错误

**解决方案：**
1. 检查 Vercel 部署日志，确认构建成功
2. 确认部署状态是 "Ready"
3. 尝试清除浏览器缓存
4. 如果还是不行，尝试重新部署

### 问题 5：图标不显示

**解决方案：**
- 这是正常的，因为某些图标服务可能无法访问
- 网站会显示首字母图标作为后备方案
- 不影响网站功能

### 问题 6：网站加载慢

**解决方案：**
- Vercel 的免费版在全球都有 CDN，速度应该很快
- 如果还是很慢，可能是网络问题
- 可以尝试清除浏览器缓存

## 📝 后续更新网站

当你修改了代码，想要更新网站时：

### 方法一：通过 Git 推送（推荐）

1. 修改代码后，在项目文件夹中执行：
   ```bash
   git add .
   git commit -m "更新描述"
   git push
   ```
2. Vercel 会自动检测到代码更新，并自动重新部署
3. 等待几分钟，网站就更新了！

### 方法二：在 Vercel 中重新部署

1. 在 Vercel 项目页面
2. 点击 **"Deployments"** 标签
3. 找到之前的部署记录
4. 点击右侧的 **"..."** 菜单
5. 选择 **"Redeploy"**

# 🔧 故障排除指南

## 无法访问网站的问题

### ✅ 检查1: 确认服务器正在运行

服务器已经在端口3000上运行（进程ID: 39128）

### 🔍 检查2: 尝试不同的访问方式

请尝试以下URL：

1. **本地访问**:
   - http://localhost:3000
   - http://127.0.0.1:3000

2. **网络访问** (如果使用 `-H 0.0.0.0`):
   - http://你的IP地址:3000
   - 例如: http://192.168.1.100:3000

### 🛠️ 解决方案

#### 方案1: 重启开发服务器

```bash
# 停止当前服务器（在终端按 Ctrl+C）
# 然后重新启动
npm run dev
```

#### 方案2: 检查防火墙

Windows防火墙可能阻止了端口3000：

1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 检查入站规则是否允许端口3000
4. 或者临时关闭防火墙测试

#### 方案3: 使用不同的端口

如果3000端口被占用，可以修改端口：

1. 编辑 `package.json`:
```json
"dev": "next dev -H 0.0.0.0 -p 3001"
```

2. 然后访问: http://localhost:3001

#### 方案4: 检查浏览器

1. 尝试不同的浏览器（Chrome、Firefox、Edge）
2. 清除浏览器缓存
3. 尝试无痕/隐私模式
4. 检查是否有浏览器扩展阻止了访问

#### 方案5: 检查是否有错误

查看终端输出，看是否有错误信息。常见错误：

- **端口被占用**: 尝试使用不同端口
- **依赖缺失**: 运行 `npm install`
- **编译错误**: 检查代码是否有语法错误

### 🆘 如果仍然无法访问

1. **查看终端输出**: 检查是否有错误信息
2. **检查Node.js版本**: 需要Node.js 18+
   ```bash
   node --version
   ```
3. **重新安装依赖**:
   ```bash
   npm install
   ```
4. **清理并重建**:
   ```bash
   rm -rf .next
   npm run build
   npm run dev
   ```

### 💡 替代方案

如果本地无法访问，可以考虑：

1. **使用Vercel部署**: 快速部署到云端
2. **使用其他端口**: 修改为8080或其他端口
3. **使用VPN/代理**: 如果有网络限制
